# Lean Flag Algebras — Blueprint

A [leanblueprint](https://github.com/PatrickMassot/leanblueprint) mathematical
blueprint and **dependency graph** for the core of this library: everything in
`LeanFlagAlgebras/FlagAlgebra/` (Razborov's flag algebras) plus a worked
application chapter on **Mantel's theorem** (the Turán density of the triangle).

- **120 nodes / 179 edges** across 8 chapters: flags, subflag densities, the flag
  algebra `Aˢ`, the unlabeling operator `⟦·⟧₀`, positivity & sum-of-squares,
  limit semantics & random homomorphisms, the computational layer, and Mantel.
- Every node is tagged with the Lean declaration that realizes it. All **151**
  `\lean{}` tags were machine-checked to exist in the compiled library
  (checkdecls-equivalent, 0 missing).

## View the dependency graph

**Online:** <https://taeyool.github.io/lean-flag-algebras/dep_graph_document.html>
(blueprint text at <https://taeyool.github.io/lean-flag-algebras/>). The site is
deployed automatically from `blueprint/web/` by
`.github/workflows/blueprint.yml` on every push to `main`.

**Locally** (no toolchain needed; the generated site is committed under
`blueprint/web/`): the interactive graph is rendered client-side (WebAssembly
graphviz), so it must be served over HTTP — opening the file directly will not
render it. From the repository root:

```bash
cd blueprint/web
python3 -m http.server 8000
```

Then open **http://localhost:8000/dep_graph_document.html** (the dependency
graph) or **http://localhost:8000/** (the blueprint text). Any static file server
works. Only rendering of the LaTeX math needs internet (MathJax CDN); the graph
itself works offline.

Node colors: **box** = definition, **ellipse** = theorem/lemma; green border =
statement formalized; light-green fill = definition done, green fill = proof
formalized, dark-green fill = the whole dependency subtree is formalized; white
interior = statement tagged but no proof sketch written in the blueprint yet.

## Rebuild from source

Sources live in `blueprint/src/` (`content.tex` inputs the per-layer chapters in
`blueprint/src/chapters/`). Requires Python with `leanblueprint`, a LaTeX
distribution, and graphviz:

```bash
pip install leanblueprint
leanblueprint web        # -> blueprint/web/   (on Windows Korean locale, prefix PYTHONUTF8=1)
```

## Re-verify the Lean declaration names

`blueprint/lean_decls` (regenerated by `leanblueprint web`) lists every tagged
declaration. With the `FlagAlgebra/` and `MantelTheorem/` modules built, checking
that each name exists is a `checkdecls`-style pass over `Lean.Environment.contains`.
